# Design Collaboration

## Current Position

Keep Product Pipeline and design separate for now, with a clean collaboration boundary.

Product Pipeline should determine product direction, target user, emotional calling, feature priorities, and behavior slices. A future designer or design module should translate those outputs into visual identity, UX flows, interface patterns, and concrete design assets.

## Product Pipeline May Produce

- Design brief.
- Target user and emotional tone.
- Brand adjectives and anti-adjectives.
- UX priorities.
- Critical workflows.
- Feature priority and non-goals.
- Open design questions.

## Design Module May Produce Later

- Brand direction.
- Interaction model.
- Wireframes or mockups.
- Visual system recommendations.
- Design acceptance notes.
- Assets or component guidance.

## Handoff Rule

If a task needs visual direction to be buildable, Product Pipeline should create a design-needed work item instead of pretending the implementation task is ready.

## Open Decision

The module design still needs Richard's call on whether v0 Product Pipeline includes a lightweight design brief internally or delegates all visual/brand shaping to a separate design module.
