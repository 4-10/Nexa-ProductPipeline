# Nexa Product Pipeline

Nexa Product Pipeline is the module that will turn human product ideas into researched, structured, developable work for Nexa.

It exists because implementation throughput can outpace backlog creation. The module should help agents guide product discovery, run research, synthesize strategy, create epics and tasks, update GitHub Projects, and leave an auditable trail in the parent KB.

## What The Module Should Eventually Do

- Intake high-level product ideas.
- Ask iterative product questions.
- Coordinate research and comparative analysis.
- Identify target market focus, differentiators, and emotional calling.
- Produce opportunity briefs, epics, stories, and tasks at the right level of detail.
- Create or update GitHub Issues and GitHub Project items.
- Guide subagents with rules, formats, audit expectations, and accountability.
- Handoff to AppGenerator/App Builder and future design collaboration when appropriate.

## What This Repo Is

This repo is the build workspace for the module, including a temporary KB and GitHub Project. It is not the final Product Pipeline runtime artifact.
